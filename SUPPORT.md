# Support

- Use GitHub Discussions for design questions, world-building ideas, and help
  using the reference client.
- Use GitHub Issues for reproducible bugs and focused feature proposals.
- Use private vulnerability reporting for security concerns; do not publish
  credentials, local Convex state, or bridge tokens in issues.

Before opening an issue, run `npm run godot:check-core` and include the failing
command, platform, Node version, Godot version, and relevant sanitized logs.
